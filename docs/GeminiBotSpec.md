Project Loa: Autonomous Documentation AgentArchitecture Spec | v1.01. Executive SummaryThis project builds an autonomous agent ("Loa Bot") that monitors crypto governance forums and GitHub repositories to maintain up-to-date protocol documentation.Core Philosophy:Code over Chat: The agent does not "chat"; it writes and executes TypeScript scripts to perform tasks.Skills over Prompts: We use the Anthropic "Agent Skills" pattern (SKILL.md directories) to give the agent procedural memory.Direct CLI: We use the gemini CLI in headless mode to leverage existing auth and quotas.

References:
Anthropic: Agent Skills https://www.anthropic.com/engineering/equipping-agents-for-the-real-world-with-agent-skills
Cloudflare: Code Mode https://blog.cloudflare.com/code-mode/
Ralph Architecture https://ghuntley.com/ralph/
McPorter (MCP to Code) https://github.com/steipete/mcporter

// The "Long-Term Memory" of a Protocol
// Stores the agent's summarized understanding of a protocol's state.
model ProtocolAgentContext {
  id                  String   @id @default(cuid())
  protocolId          String   @unique
  technicalSummary    String?  @db.Text 
  governanceState     String?  @db.Text 
  lastProcessedPostId Int?     @default(0) // Bookmarking for the Forum Mapper
  updatedAt           DateTime @updatedAt

  protocol            Protocol @relation(fields: [protocolId], references: [id])

  @@map("protocol_agent_context")
  @@schema("agent")
}

// "Notebook" for tracking key actors/delegates
// Stores observations about specific users derived from forum activity.
model EntityObservation {
  id               String   @id @default(cuid())
  entityIdentifier String   // Link to TokenHolder.address or ForumAuthor.username
  content          String   @db.Text 
  confidenceScore  Int      @default(50) 
  createdAt        DateTime @default(now())

  @@index([entityIdentifier])
  @@map("entity_observations")
  @@schema("agent")
}

# Governance Watchdog

## Objective
Read pending forum posts, synthesize the changes to the protocol, and log observations about key delegates.

## Tooling
* **DB Access:** `import * as DbTool from '../../tools/db';`
* **Key Tables:** `governance.forum_posts`, `agent.protocol_agent_context`, `agent.entity_observations`.

## Workflow
1.  **Fetch State:** Query `agent.protocol_agent_context` to get `lastProcessedPostId`.
2.  **Fetch Batch:** Query next 20 posts from `governance.forum_posts` (joined with topics).
    * *Tip:* Join `discourse_topics` to get the thread title.
    * *Tip:* Use `cookedContent` for the post body.
3.  **Analyze & Commit:**
    * **State Change:** If a post implies a parameter change, new feature, or strategic shift, UPDATE `governanceState`.
    * **Entity Observation:** If a user (check `authorUsername`) expresses a strong view, INSERT into `agent.entity_observations`.
    * **Advance Cursor:** UPDATE `lastProcessedPostId` to the max ID seen.

## Output
**Return ONLY valid, executable TypeScript code.** Do not use markdown blocks.

# Repo Walker

## Objective
Map the file structure of a protocol to understand its technical architecture.

## Tooling
* **Repo Access:** `import * as RepoTool from '../../tools/repo';`

## Workflow
1.  **Clone:** Use `RepoTool.clone(url)`.
2.  **Map:** Use `RepoTool.list_files()` to find the `contracts/` or `src/` directory.
3.  **Read:** Read `README.md` and the largest `.sol` file to understand the core logic.
4.  **Report:** Write a summary to `agent.protocol_agent_context.technicalSummary`.

## Output
**Return ONLY valid, executable TypeScript code.**
6. Implementation (agent.ts)The controller that connects the CLI to the Tools using the "Ralph" loop pattern.import { exec } from "child_process";
import { promisify } from "util";
import * as fs from "fs";
import * as path from "path";
import { runInVm } from "./sandbox"; 

// Tool Imports (Generated by mcporter)
import * as DbTool from "./tools/db";
import * as RepoTool from "./tools/repo";

const execAsync = promisify(exec);

async function callGemini(skillPath: string, taskContext: any) {
  const skillContent = fs.readFileSync(skillPath, "utf-8");
  
  // Construct the prompt file
  const prompt = `
    ${skillContent}
    
    CURRENT TASK CONTEXT:
    ${JSON.stringify(taskContext)}
  `;
  
  const tempFile = path.join(__dirname, ".temp_prompt.txt");
  fs.writeFileSync(tempFile, prompt);

  try {
    // Pipe to Gemini CLI in Headless Mode
    // Adjust command flags based on your specific CLI version
    const { stdout } = await execAsync(`cat ${tempFile} | gemini prompt`);
    return stdout.replace(/```typescript|```/g, "").trim();
  } finally {
    if (fs.existsSync(tempFile)) fs.unlinkSync(tempFile);
  }
}

async function ralphLoop(workItem: any) {
  console.log(`[Loa] Starting work on ${workItem.id}...`);
  
  const skillMap = {
    "forum_update": ".gemini/skills/governance/SKILL.md",
    "new_protocol": ".gemini/skills/onboarding/SKILL.md"
  };

  let attempts = 0;
  let lastError = null;

  while (attempts < 3) {
    try {
      const context = lastError ? { ...workItem, error: lastError } : workItem;
      
      console.log(`   Attempt ${attempts + 1}: Generating Code...`);
      const code = await callGemini(skillMap[workItem.type], context);

      console.log(`   Executing Code...`);
      // "Code Mode": Execute the generated script in a sandbox with access to tools
      await runInVm(code, { DbTool, RepoTool, console }); 
      
      console.log("   ✅ Success");
      return;
    } catch (e) {
      console.error(`   ❌ Error: ${e.message}`);
      lastError = e.message;
      attempts++;
    }
  }
  console.error("   💀 Failed after max attempts.");
}
